# 智能简历分析平台

## 模块一：简历上传和解析

1. 简历上传：支持拖拽上传和点击上传，仅限 PDF 格式，支持批量上传，至少同时上传 5 份，需要展示上传进度和状态。**加分项**：上传过程中展示实时预览（PDF 首页缩略图）
2. 支持 PDF 内容解析，提取文本内容，需要兼容多页简历。
3. 对提取的简历文本进行清洗和结构化处理，使用 AI 模型提取出简历中的信息：
| 类别 | 字段 | 是否必须 |
| --- | --- | --- |
| **基本信息** | 姓名、电话、邮箱、所在城市 | ✅ 必选 |
| **教育背景** | 学校、专业、学历、毕业时间 | ✅ 必选 |
| **工作经历** | 公司名称、职位、时间段、工作内容摘要 | ✅ 必选 |
| **技能标签** | 技术栈、工具、语言等（以标签形式提取） | ✅ 必选 |
| **项目经历** | 项目名称、技术栈、个人职责、项目亮点 | ⭐ 加分项 |

+ AI 提取过程通过 **SSE（Server-Sent Events）流式返回**，前端逐步渲染提取结果
+ **加分项**：支持用户在前端手动修正 AI 提取的信息

## 模块二：岗位匹配和智能评分

1. 提供岗位需求配置界面（JD 编辑器），支持输入岗位描述、必备技能、加分技能
2. 调用 AI 模型对简历与岗位需求进行匹配分析，返回： 
    - **综合匹配度评分**（0-100 分）
    - **各维度子评分**：技能匹配度、经验相关性、教育背景契合度
    - **AI 评语**：一段文字说明该候选人的优势与不足
+ 评分结果以 **可视化图表** 展示（如雷达图 / 柱状图 / 环形进度条）
+ **加分项**：支持多个 JD 同时对比评分

## 模块三：候选人管理面板
1. 实现候选人列表页，支持： 
    - **表格视图 / 卡片视图** 切换
    - 按评分、上传时间、技能标签 **排序与筛选**
    - 关键字搜索（姓名、技能、学校等）
    - 分页或虚拟滚动
2. 点击候选人进入 **详情页**，展示完整的解析信息、评分详情、原始 PDF 预览
3. 支持候选人状态管理（待筛选 → 初筛通过 → 面试中 → 已录用 / 已淘汰），状态变更需有视觉反馈
4. **加分项**：支持候选人对比功能（选择 2-3 人，并排对比各维度评分）

以上是业务需求，以下是工程质量要求：

+ 使用 **组件化架构**，合理拆分组件粒度，体现复用性和可维护性
+ 页面整体需具备 **良好的视觉设计和交互体验**（不接受纯原生 HTML 样式）
+ 实现 **响应式布局**，适配桌面端（≥1280px）
+ 全局错误处理与 Loading 状态管理
+ **加分项**： 
    - 暗色 / 亮色主题切换
    - 骨架屏（Skeleton）加载态
    - 键盘快捷键支持
    - 动画过渡效果

技术栈：
1. 使用 pnpm workspace 构建 monorepo 架构，用同一个代码仓库管理前后端。
2. 前端使用 Next.js + Typescript + Tailwind CSS + Zustand + SWR
3. 后端使用Fastify+postgresql（neon的数据库），调用大模型使用 vercel 的 ai sdk。
4. 前后端都统一部署到 vercel 上。
